# Webpack Configuration

Owner: Frontend Platform Team
Reviewers: Theme Team, QA
Last Updated: 2026-06-11
Last Reviewed: 2026-06-11
Status: Approved

The Webpack configuration lives at `webpack.config.js` in the repo root. It exports a function that the FDK CLI invokes with environment options (`isLocal`, `isHMREnabled`, asset base paths, etc.) and merges into its own base build config.

## Entry points

The config defines a single entry, `themeBundle`, pointing at `theme/index.jsx`. Section entries are generated by the FDK CLI build, not by this file.

Section chunking is enabled (`fdk_feature.enable_section_chunking: true` in `package.json`), so each section in `theme/sections/` is emitted as a separate chunk for lazy loading on the platform.

## Output

| Directory | Contents |
|---|---|
| `dist/` | Main Webpack build artifacts |
| `dist_sections/` | Extracted section chunks |

Both are removed by `npm run clean` before every build.

## Loaders

| Loader | File types handled | Notes |
|---|---|---|
| `babel-loader` | `.js`, `.jsx` (in `theme/`) | `@babel/preset-env` + `@babel/preset-react`; `react-refresh/babel` added for local HMR builds |
| `babel-loader` | `.ts`, `.tsx` | Adds `@babel/preset-typescript` and `@loadable/babel-plugin` |
| `css-loader` | `.css` | Global styles (`modules: false`) |
| `less-loader` + `css-loader` | `.less` | CSS Modules by default; `.global.less` files compile as global styles |
| `@svgr/webpack` | `.svg` | Import SVGs as React components |
| `asset/resource` | Images (`.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`) | Emitted to `assets/images/`, public path depends on local vs CDN |
| `asset/resource` | Fonts (`.ttf`, `.otf`, `.woff`, `.woff2`) | Emitted to `assets/fonts/` |
| `file-loader` | Fonts (`.ttf`, `.eot`, `.woff`, `.woff2`) | Legacy font rule |

## Plugins

| Plugin | Purpose |
|---|---|
| `MiniCssExtractPlugin` | Extracts CSS into separate files (`[name].[contenthash].css` in production) |
| `NodeJSPolyfill` (custom, `./plugin`) | Injects the `polyfill.js` snippet (Node.js globals) into bundles |
| `DotenvWebpack` | Injects `.env` variables — local builds only |

`react-hydration-overlay` (hydration mismatch debugging) and `webpack-bundle-analyzer` are imported/wired but currently commented out in `webpack.config.js`.

## Optimization

| Optimization | Config |
|---|---|
| CSS minification | `CssMinimizerPlugin` added to minimizers (active in production mode) |
| JS minification | Webpack default Terser in production mode |
| Code splitting | `@loadable/component` + `@loadable/babel-plugin` |
| Externals | `react-i18next` and `i18next` mapped to platform-provided globals (`i18nReact`, `i18next`) |

## Build modes

| Command | Mode | Notes |
|---|---|---|
| `npm run build` | `production` | Cleans, then minified build |
| `npm run build:dev` | `local` (watch) | Cleans, unminified, watches for changes |
| `npm run build:local` | `local` | Cleans, unminified single build |
| `npm run analyze` | `production` | Runs build with `ANALYZE=true` |

## Bundle analysis

```bash
npm run analyze
```

The config reads `process.env.ANALYZE === "true"`, but the `BundleAnalyzerPlugin` block inside it is currently commented out — uncomment it (and its import) in `webpack.config.js` to get the interactive treemap report.

Key things to check when analyzing:
- Individual section chunk sizes (should be < 50KB each for fast lazy loading).
- Whether shared dependencies are properly deduplicated into common chunks.
- Whether SVG imports are tree-shaken (avoid importing large SVG libraries wholesale).

## Less module conventions

Less files are CSS Modules by default:

```jsx
import styles from "./my-component.less";

// Usage
<div className={styles.container}>...</div>
```

To write global (unscoped) styles, name the file with a `.global.less` suffix. Plain `.css` files are always global (`modules: false`). In local builds, generated class names include the file path and local name for easier debugging; production uses short hashes.

## Known configuration gaps

- No source maps in production builds.
- `.ts`/`.tsx` are resolvable and have a Babel rule, but the theme source is plain JS/JSX.
- Bundle analyzer and hydration overlay plugins are present in `devDependencies` but commented out in the config.
